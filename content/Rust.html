main.rs

fn main() {
    for i in 1..11 {
        print!("第 {} 次說 {}!\n", i, "Hello, world");
    }
}

rs.bat

@echo off
cls
rustc %* --color always 2>&1 |

利用 rs main.rs 可以編譯出 main.exe

If it is compiled and run as before, it prints: "Hello, world!"

Notice that the newly added line contains eight syntax items, aka “tokens”. Let’s examine them:

  1. print: It is the name of a macro defined in the Rust standard library.
  2. !: It specifies that the preceding name indicates a macro. Without
    such a symbol, print would instead indicate a function. There is no
    such function in the Rust standard library, and so you would get a
    compilation error. A macro is a thing similar to a function - it’s some
    Rust code to which a name is associated. By using this name, you ask
    to insert such code in this point.
  3. (: It starts the list of arguments of the macro.
  4. ": it starts the literal string.
  5. Hello, world!: It is the content of the literal string.
  6. ": It ends the literal string.
  7. ): It ends the list of the arguments of the macro.
  8. ;: It ends the statement.

透過 C:

利用 cargo new rust_python 建立 rust_python 目錄及專案檔案內容.

rust_python 目錄檔案:

.git

src 目錄

.gitignore

Cargo.toml

Cargo.toml

[package]
name = "rust_python"
version = "0.1.0"
edition = "2021"

[lib]
name = "square"
crate-type = ["dylib"]

# See more keys and their definitions at https://doc.rust-lang.org/cargo/reference/manifest.html

[dependencies]

src\lib.rs

#[no_mangle]
pub extern "C" fn square(x: i32) -> i32 {
    x*x
}

編譯 square.dll

進入 rust_python 目錄後, 執行 cargo build

call_rust.py

import cffi

ffi = cffi.FFI()
s = 'int square(int);'
ffi.cdef(s)
C = ffi.dlopen("./rust_python/target/debug/square.dll")

print(C.square(9))

執行 call_rust.py 後得到 81.
